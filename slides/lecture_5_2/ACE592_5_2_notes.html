1 Functional equations

1.1 Problem type: Functional equations

In these problems, our solution is not a value but a function

Formally, we look for f such that Tf = 0, where T is an operator that maps a vector space of functions into itself

  • We can write an equivalent functional fixed-point equation f = Tf

. . .

Functional equations are hard because they impose an infinite number conditions, i.e., the value of f at an infinite number of points - Solutions with closed-form expressions are an exception!

Problems like this arise frequently in dynamic optimization and strategic interaction/game theory

1.2 Functional equations general definition

In a classic formulation of functional equations, we look for a function f such that

g\left(x, f(x) \right) = 0, \; \forall x \in [a, b]

. . .

Examples

  • Differential equations
    • g(x, f(x)) \equiv f^\prime(x) + f(x) + 2 = 0
  • Bellman equations
    • g(x, V(x)) \equiv V(x) - \left\{\max_z u(x) + \beta V(\phi(x)) \right\} = 0
  • Monopolist supply S(p) with marginal cost MC and facing demand D(p)
    • g(p, S(p)) \equiv p + \frac{S(p)}{D^\prime(p)} - MC(S(p)) = 0

1.3 Numerically solving functional equations

Problems with functional equations are typically quite hard to solve and only admit closed-form solutions in special cases

We can use numerical function approximation tools to make the problem tractable!

Key idea. Instead of looking for f in an infinite-dimensional function space, we look for its approximant \hat{f} in a finite-dimensional space

1.4 Numerically solving functional equations

Once again, we use approximants with a linear combination of basis functions

f(x) \approx \hat{f} = \sum^{n}_{j=1} \phi_j(x) c_j = \phi(x)c

So the conditions on the original functional equation becomes

g(x, \phi(x)c) \approx 0, \; \forall x \in [a, b]

where g(x, \phi(x)c) can be regarded as a residual, which we want to make as close to zero as possible for a good approximation

1.5 Collocation method

The general approach to numerically solve problems of this form is called the Collocation method

Instead of solving an infinite-dimensional problem, we look for a finite vector of basis coefficients c such that the residual is 0 at n nodes

g(x_i, \phi(x_i)c) = 0, \; \forall i = 1, 2, \dots, n

This is a n-dimensional (nonlinear) rootfinding problem! We can use the methods from Unit 3 to solve it

1.6 Collocation method

How do we assess the quality of our approximation?

The common approach is to evaluate the residual g(x, \phi(x)c) over a fine grid (not Chebyshev nodes) and look for the highest absolute value

  • If the largest residual is above some tolerance level we defined, we can repeat the process with more nodes and basis functions to improve accuracy

1.7 Collocation method in practice

We will cover the collocation method in practice with two examples:

  1. Solving a static Cournot oligopoly model
  2. Solving a dynamic intertemporal utility maximization problem

2 Collocation example 1: Cournot oligopoly

2.1 Cournot oligopoly: Setup

This example is based on the one in Miranda & Fackler (2002), Ch. 6.8

Standard profit maximization: \max_q qp(q) - MC(q)

In a competitive market (price-takers), p is unaffected by q, so the FOC simplifies to p - MC(q) = 0

. . .

In an oligopoly, a firm’s output influences price, so the FOC becomes: p + q\frac{dp}{dq} - MC(q) = 0

2.2 Cournot oligopoly: Supply function

Using the Inverse Function Theorem, we can rewrite the FOC as p + \frac{q}{D^\prime(p)} - MC(q) = 0

. . .

We want to characterize the firm’s effective supply function S(p), the optimal supply for any given p

S(p) satisfies the functional equation:

g(p, S(p)) \equiv p + \frac{S(p)}{D^\prime(p)} - MC(S(p)) = 0

2.3 Cournot oligopoly: Analytical solution

For simple functional forms, we can solve analytically

With MC(q) = c and D(p) = p^{-\eta}:

p - \frac{S(p)}{\eta p^{-\eta-1}} - c = 0 \implies S(p) = \eta(p-c)p^{-\eta-1}

. . .

With m identical firms, equilibrium satisfies mS(p) = D(p), which yields: p = \left(\frac{1}{1 - \frac{1}{\eta m}}\right)c

As m \rightarrow \infty, we approach perfect competition: p = c

2.4 When simple functional forms are not available

More realistic cost functions may not yield tractable solutions

Suppose MC(q) = \alpha q^{1/2} + q^2. The FOC becomes: p - \frac{p^{\eta+1}}{\eta}S(p) - \alpha [S(p)]^{1/2} - [S(p)]^2 = 0

There is no closed-form solution for S(p)!

. . .

Solution: Use the collocation method to numerically find \hat{S}(p) over p \in [a, b]

2.5 Setting up the collocation problem

We look for \hat{S}(p) = \phi(p)c (a Chebyshev polynomial approximant) such that g(p_i, \hat{S}(p_i)) \equiv p_i - \frac{p_i^{\eta+1}}{\eta}\hat{S}(p_i) - \alpha [\hat{S}(p_i)]^{1/2} - [\hat{S}(p_i)]^2 = 0

at n Chebyshev nodes p_1, \dots, p_n

. . .

This is a rootfinding problem: find a vector c such that g(c) = 0

  • We’ll use \alpha = 1, \eta = 1.5, domain p \in [0.1, 3], and n = 25 nodes

2.7 Implementation: Evaluating the polynomial

We need a function to evaluate \hat{f}(x) = \sum_{j=1}^n c_j \phi_j(x) at any x \in [a, b]

Code 
function eval_chebpoly(c, n, a, b, x)
    domain = [b, a]  # upper bound first (ChebyshevApprox convention)
    y = similar(x)
    for i in 1:length(x)
        y[i] = chebyshev_evaluate(c, [x[i]], n - 1, domain)
    end
    return y
end;

2.8 Implementation: Nodes and residual function

Define the Chebyshev nodes and the residual function g(c) that we want to find a root of

Code 
# Chebyshev nodes
nodes = ChebyshevApprox.nodes(n, :chebyshev_nodes, [b, a]).points
# The function to calculate the residual at the nodes given a vector of coefficients c
function resid(c, ps)
    S_hat = eval_chebpoly(c, n, a, b, ps)
    ps - ps.^(η+1) ./ η .* S_hat - α .* sqrt.(S_hat) - S_hat.^2
end

g(c) = resid(c, nodes);

2.9 A problem with unconstrained polynomials

When we attempt to solve g(c) = 0 with Newton’s method:

Code

DomainError with -0.998...:
sqrt will only return a complex result if called with a
complex argument. Try sqrt(Complex(x)).

. . .

Newton’s method took a step where \hat{S}(p_i) < 0, but \sqrt{\hat{S}} requires \hat{S} \geq 0

The polynomial \hat{S}: [a,b] \rightarrow \mathbb{R} does not automatically respect the economic constraint S \geq 0

3 Collocation example 2: Dynamic optimization

3.1 Cake-eating problem: Setup

A canonical infinite-horizon consumption vs. savings problem: eat the cake today or save it for tomorrow?

An agent starts with a cake of size W_0 and decides how much to eat each period:

  • Consumption: x_t \in [0, W_t]
  • Transition: W_{t+1} = W_t - x_t
  • For simplicity, the cake neither grows nor depreciates

. . .

The agent maximizes lifetime utility: \max_{\{x_t\}_{t=0}^\infty} \sum_{t=0}^\infty \beta^t u(x_t)

with CRRA utility u(x) = \frac{x^{1-\sigma}}{1-\sigma}, RRA \sigma = 1.5 and \beta = 0.95

3.2 Quick review: From sequence to recursion

The sequence problem above has infinitely many choice variables, one per period

Key insight (Bellman, 1957): any tail of an optimal plan is itself optimal

This principle of optimality lets us rewrite the problem recursively in terms of a value function V(W), the maximized lifetime utility starting from state W:

V(W) = \max_{0 \leq x \leq W} \left\{u(x) + \beta\, V(W - x)\right\}

This is a functional equation for V, which we will solve using the collocation method

3.3 Bellman equation as a functional equation

We can express the Bellman equation in fixed-point form V = TV, where T is the Bellman operator:

(TV)(W) \equiv \max_{0 \leq x \leq W} \left\{u(x) + \beta\, V(W - x)\right\}

Equivalently, define the Bellman residual: g(W, V(W)) \equiv V(W) - \max_{0 \leq x \leq W} \left\{u(x) + \beta\, V(W - x)\right\} = 0

The goal is to find V such that g(W, V(W)) = 0 for all W in the domain

3.4 An analytical benchmark

CRRA cake-eating admits a closed-form solution we can use to validate our numerical method

Guess V(W) = A \cdot \frac{W^{1-\sigma}}{1-\sigma} and verify with the FOC u'(x) = \beta V'(W - x)

. . .

This yields a constant savings rate policy: x^*(W) = \left(1 - \beta^{1/\sigma}\right) W, \qquad A = \left(1 - \beta^{1/\sigma}\right)^{-\sigma}

For \sigma = 1.5, \beta = 0.95: x^*(W) \approx 0.0336\, W, i.e., eat about 3.4% of the cake each period

3.5 Why solve this numerically?

The closed form depends on a lucky combination of CRRA utility and a deterministic linear cake transition

But our luck ends as soon as we add realistic features:

  • Stochastic returns or shocks to the cake
  • Borrowing or short-selling constraints
  • Non-CRRA preferences (habits, reference dependence)
  • Multiple state variables

The collocation steps we build here generalizes for these cases, and all we’d need to change are the the residual functions

3.6 Collocation setup

Approximate the value function with Chebyshev polynomials:

V(W) \approx \hat V(W; c) = \sum_{j=1}^n c_j\, \phi_j(W) = \phi(W) c

At each Chebyshev node W_i \in [\underline W, \overline W], the Bellman residual is:

g_i(c) = \phi(W_i) c - \max_{0 \leq x \leq W_i} \left\{u(x) + \beta\, \phi(W_i - x)\, c\right\}

We seek c \in \mathbb{R}^n such that g_i(c) = 0 for i = 1, \dots, n

3.7 Nested structure of the problem

The collocation problem has two nested layers:

  1. Outer problem: find coefficients c that make all n residuals zero
  2. Inner problem: at each node W_i, optimize over consumption x given the current approximant \hat{V}

3.8 Solution strategy

Two common solution strategies:

  • Function iteration (a.k.a. fitted value iteration): iterate c^{(k+1)} as the basis-coefficient projection of T\hat V(\cdot; c^{(k)})
    • Robust, but only linear convergence
  • Newton on the residual system g(c) = 0
    • Quadratic convergence near the root, but more sensitive to the initial guess

We will use function iteration, which mirrors classical Bellman value iteration idea

3.9 Implementation roadmap

Function iteration on coefficients:

  1. Choose domain [\underline W, \overline W], number of nodes n, and Chebyshev basis
  2. Initialize coefficients c^{(0)}
  3. Repeat until \|c^{(k+1)} - c^{(k)}\| < \text{tol}:
    1. For each node W_i, solve the inner problem V_i = \max_{0 \leq x \leq W_i} u(x) + \beta\, \hat V(W_i - x; c^{(k)})
    2. Update coefficients c^{(k+1)} by interpolating the new node values
  4. Validate against the analytical benchmark

We must keep W bounded away from zero, since u'(x) \to \infty as x \to 0 under CRRA

3.13 Implementation: Inner optimization

At each node W_i, find consumption that maximizes the Bellman RHS, given current coefficients

Code 
function bellman_rhs(c_coef, Wi)
    # Note that Optim requires the solution to be a vector, even if it's a scalar
    obj(x) = -(u(x[1]) + β * V_hat(c_coef, Wi - x[1]))
    # Use box constrained optimization to enforce 0 ≤ x ≤ Wi and midpoint as initial guess
    res = Optim.optimize(obj, [W_min], [Wi], [(Wi+W_min)/2], Fminbox(BFGS()))
    x_opt = res.minimizer[1]
    V_val = -res.minimum
    return V_val, x_opt
end

3.14 Implementation: One Bellman update on coefficients

Given c^{(k)}, evaluate the Bellman RHS at every node and refit the polynomial

Code 
function bellman_update(c_coef)
    # Initialize vectors
    V_vals = zeros(n); pol_vals = zeros(n)
    # Solve the inner problem at each node
    for i in 1:n
        V_vals[i], pol_vals[i] = bellman_rhs(c_coef, nodes[i])
    end
    # Recalculate Chebyshev coefficients
    new_coef = chebyshev_weights(V_vals, nodes, n-1, domain)
    return new_coef, V_vals, pol_vals
end

3.15 Implementation: The outer iteration

Code 
function solve_collocation(; tol = 1e-6, maxit = 1000)
    # Warm-start assuming all consumption at initial period
    V_init = u.(nodes)
    c_coef = chebyshev_weights(V_init, nodes, n-1, domain)
    # To avoid infinite looping
    for iter in 1:maxit
        new_coef, V_vals, pol_vals = bellman_update(c_coef)
        diff = maximum(abs.(new_coef - c_coef))
        if diff < tol
            println("Converged in $iter iterations (Infty norm = $(round(diff, sigdigits=3)))")
            return new_coef, V_vals, pol_vals
        end
        c_coef = new_coef
    end
    println("Did not converge in $maxit iterations")
    return c_coef, zeros(n), zeros(n)
end;

3.17 Solution: Simulation

Let’s simulate three sample trajectories of consumption for different initial cake sizes. First, we need a function to simulate the trajectory given the coefficients of the value function approximant

Code 
function simulate_trajectory(c_coef, W0, T)
    W_traj = zeros(T); x_traj = zeros(T)
    W_traj[1] = W0
    for t in 1:T-1
        x_traj[t] = bellman_rhs(c_coef, W_traj[t])[2]
        W_traj[t+1] = W_traj[t] - x_traj[t]
    end
    x_traj[T] = bellman_rhs(c_coef, W_traj[T])[2]
    return W_traj, x_traj
end

3.23 Wrap-up

  • Functional equations impose infinitely many conditions on an unknown function
  • The collocation method reduces this to a finite-dimensional problem by:
    1. Approximating f with a finite basis \hat f = \phi c
    2. Imposing the conditions only at n well-chosen nodes
  • Solution quality is judged by residuals on a fine grid, not at the nodes
  • The same recipe scales up to richer dynamic problems (stochastic shocks, multiple state variables, constraints); only the residual function changes
