Recall the model: specification

Utility for individual \(i\), draw \(s\):

\[ V_{ib}^{(s)} = \beta_{b,0} + \beta_{i,p}^{(s)} p_b + \beta_{i,sb}^{(s)} \text{stops}_b + \beta_{b,\text{den}} d + \beta_{b,\text{hh}} h + \beta_{b,\text{ho}} o \]

\[ V_{it}^{(s)} = \beta_{t,0} + \beta_{i,p}^{(s)} p_t + \beta_{i,st}^{(s)} \text{stops}_t + \beta_{t,\text{den}} d + \beta_{t,\text{hh}} h + \beta_{t,\text{ho}} o \]

where \(\beta_{i,p}^{(s)} = -\exp(b_p + s_p \eta_{i,p}^{(s)})\), \(\beta_{i,sb}^{(s)} = b_{sb} + s_{sb} \eta_{i,sb}^{(s)}\), \(\beta_{i,st}^{(s)} = b_{st} + s_{st} \eta_{i,st}^{(s)}\).

Recall the model: parameter values

Parameter vector \(\theta\) (14 elements):

Index Symbol Meaning
1–4 \(\beta_{b,0},\,\beta_{b,\text{den}},\,\beta_{b,\text{hh}},\,\beta_{b,\text{ho}}\) Bus fixed: intercept, density, hhsize, homeown
5–8 \(\beta_{t,0},\,\beta_{t,\text{den}},\,\beta_{t,\text{hh}},\,\beta_{t,\text{ho}}\) Train fixed: intercept, density, hhsize, homeown
9–10 \(b_p,\,s_p\) Log-normal price: location, scale
11–12 \(b_{sb},\,s_{sb}\) Normal bus stops: mean, std dev
13–14 \(b_{st},\,s_{st}\) Normal train stops: mean, std dev

Compare to 8.3b’s restricted model (11 parameters): we have the same 8 alternative-specific fixed coefficients plus 6 random-coefficient parameters: location and scale for each of the three random coefficients.

2. Maximum Simulated Likelihood

Simulation draws

  • As it will soon become clear, we will push the boundaries of computation resource in this exercise, so we need to squeeze every bit of efficiency we can get.

  • We will run quasi-Monte Carlo integration with Halton sequences using the QuasiMonteCarlo package. This sequence has better convergence properties for numerical integration, meaning they require fewer draws for the same accuracy compared to default pseudo-random generator in Julia (Xoshiro256)

  • Let’s illustrate how Halton draws make more efficient use of draws compared to pseudo-random numbers.

Pseudo-random vs. Halton draws

The left panel shows pseudo-random draws in the unit square, which can have clusters and gaps. The right panel shows Halton draws, which are more evenly spread out, leading to better coverage of the integration space and faster convergence of the simulated likelihood.

Generating Halton Draws for \(\eta\)

We draw \(\eta\)’s once, outside the optimizer, and hold them fixed: this is the common random numbers technique, which smooths the simulated log-likelihood surface. We need three matrices. Let’s write a function to generate Halton draws from \(U([0,1]^3)\) and maps to standard normal distributions using the inverse CDF transformation.

# N is number of units, S is number of simulation draws
function drawHalton_η(N, S) 
    # Draw each dimension as a vector of length N*S
    ηs = QuasiMonteCarlo.sample(N * S, 3, RandomizedHaltonSample())
    # Apply inverse CDF transformation to get standard normal draws
    ηs = quantile.(Normal(), ηs) # 3 × (N*S) matrix of standard normal draws
    # Then, reshape into N × S matrices for each parameter
    η_p =  reshape(ηs[1, :], N, S) # price draws
    η_sb = reshape(ηs[2, :], N, S) # bus stops draws
    η_st = reshape(ηs[3, :], N, S) # train stops draws
    return η_p, η_sb, η_st
end;

Simulated log-likelihood function

For individual \(i\) and draw \(s\), the log-probability of the observed choice is

\[ l_i^{(s)} = d_{ib} V_{ib}^{(s)} + d_{it} V_{it}^{(s)} - \log\!\left(1 + e^{V_{ib}^{(s)}} + e^{V_{it}^{(s)}}\right) \]

The simulated log-probability averages these over \(S\) draws using the log-sum-exp identity:

\[ \log\!\left(\frac{1}{S}\sum_s e^{l_i^{(s)}}\right) = \underbrace{\log\!\sum_s e^{l_i^{(s)}}}_{\text{log-sum-exp}} - \log S \]

Simulated log-likelihood function

function neg_msl(θ; X_b=X_b, X_t=X_t, d_b=d_b, d_t=d_t, η_p=η_p, η_sb=η_sb, η_st=η_st)
    # Unpack θ
    β_b_fix = θ[1:4]    # bus fixed:   intercept, density, hhsize, homeown
    β_t_fix = θ[5:8]    # train fixed: intercept, density, hhsize, homeown
    b_p_loc  = θ[9];   s_p_loc  = θ[10]   # log-normal price
    b_sb_loc = θ[11];  s_sb_loc = θ[12]   # normal bus stops
    b_st_loc = θ[13];  s_st_loc = θ[14]   # normal train stops
    # Create a matrix to store the log-likelihood values
    N_r = length(d_b)
    S = size(η_p, 2)  # number of simulation draws
    logl = zeros(N_r, S)
    for s in 1:S
        # Individual-specific coefficients for this draw
        β_p_s  = -exp.(b_p_loc  .+ s_p_loc  .* η_p[:,  s])   # N-vector, always < 0
        β_sb_s =       b_sb_loc .+ s_sb_loc .* η_sb[:, s]    # N-vector
        β_st_s =       b_st_loc .+ s_st_loc .* η_st[:, s]    # N-vector
        # X_b / X_t columns: [1, price, stops, density, hhsize, homeown]
        V_ib = (X_b[:, 1] .* β_b_fix[1]
             .+ X_b[:, 2] .* β_p_s           # price_bus   × β_price_i
             .+ X_b[:, 3] .* β_sb_s          # stops_bus   × β_bstops_i
             .+ X_b[:, 4] .* β_b_fix[2]      # density
             .+ X_b[:, 5] .* β_b_fix[3]      # hhsize
             .+ X_b[:, 6] .* β_b_fix[4])     # homeown
        V_it = (X_t[:, 1] .* β_t_fix[1]
             .+ X_t[:, 2] .* β_p_s           # price_train × β_price_i
             .+ X_t[:, 3] .* β_st_s          # stops_train × β_tstops_i
             .+ X_t[:, 4] .* β_t_fix[2]      # density
             .+ X_t[:, 5] .* β_t_fix[3]      # hhsize
             .+ X_t[:, 6] .* β_t_fix[4])     # homeown
        log_denom = log.(1.0 .+ exp.(V_ib) .+ exp.(V_it))
        logl[:, s]   = d_b .* V_ib .+ d_t .* V_it .- log_denom
    end
    # Taking the log of the mean from S draws using log-sum-exp (columns)
    sim_log_prob = log.(sum(exp.(logl), dims=2)) .- log(S)  # N × 1 vector of log individual log-likelihoods
    return -sum(sim_log_prob) # Then return the sum over individuals (rows)
end;

Starting values

In MSL, good starting values are crucial for convergence and speed up your estimation. A common approach is to start with the MNL estimates for the fixed coefficients and small positive values for the scale parameters. Let’s use the coefficients we estimated in 8.3b as starting values for the fixed part of the mixed logit.

For the scale parameters (\(s_p\), \(s_{sb}\), \(s_{st}\)) it’s very important to start at a positive value, not zero! Zero is a saddle point where \(\partial l/\partial s = 0\) and derivative-based methods can crash.

θ_hat_MNL = [-1.264940, 0.477165, -0.266855, -0.421171, # bus fixed: intercept, density, hhsize, homeown
             -0.750502, 0.746791, -0.127614, -0.237695, # train fixed: intercept, density, hhsize, homeown
             -0.928098, 0.1, # b_p,  s_p  (s_p  > 0!)
              0.101601, 0.1, # b_sb, s_sb (s_sb > 0!)
              0.219426, 0.1]; # b_st, s_st (s_st > 0!)

Note: In practice, don’t forget to try different initial values to assert your optimizer is finding a global maximum, not a local one. With 14 parameters, the likelihood surface can be weird with some local maxima.

We can’t have negative SD!

In this case, it’s actually not a big deal because of how we setup the model: the random coefficient is \(\beta_{i,k} = b_{k} + s_{k} \eta_{i,k}\), so \(s_{k} < 0\) just means the distribution of \(\beta_{i,k}\) is flipped around its mean \(b_{k}\). The likelihood is symmetric in \(s_{k}\), so the optimizer can find either the positive or negative solution. In practice, we can just take the absolute value of \(s_{k}\) for interpretation and inference.

Asymptotic standard errors

We compute the Hessian of the simulated log-likelihood at \(\hat\theta\) to obtain the asymptotic variance \(\hat V = (-\nabla^2 \hat l)^{-1}\).

Here, we use FiniteDiff because autodiff does not work; we’d need to reprogram our MSL functions to be compatible with ForwardDiff. With finite difference we lose accuracy but are faster.

t_hess = time()
H = FiniteDiff.finite_difference_hessian(neg_msl, θ_hat)
println("Hessian done in $(round(time()-t_hess, digits=1)) seconds")
V_hat = inv(H)
asym_SE = sqrt.(diag(V_hat))

3. Bootstrap standard errors

Bootstrap design

We resample decision-makers with replacement: in this cross-section one row = one individual, but this distinction matters in panels or clustered data. We use:

  • \(B = 100\) bootstrap replicates
  • \(S = 50\) simulation draws inside each bootstrap (this is too small, but we need it to run before the semester ends!)
  • Initial guesses are always \(\hat\theta\) to speed up
B      = 100
S_boot = 50
function neg_msl_boot(θ, idx, η_p_boot, η_sb_boot, η_st_boot)
    neg_msl(θ; X_b = X_b[idx, :], X_t = X_t[idx, :], 
        d_b = d_b[idx], d_t   = d_t[idx],
        η_p = η_p_boot[idx, :],  η_sb = η_sb_boot[idx, :],
        η_st = η_st_boot[idx, :])
end;

Bootstrap loop

THIS CAN TAKE A LOT OF TIME! I will use a trick to parallelize and speed up a bit.

boot_estimates = zeros(B, 14)
t_boot = time()
Threads.@threads for b in 1:B # This is forcing parallel processing
    idx   = rand(1:N_r, N_r) # Random resample
    η_p_boot, η_sb_boot, η_st_boot = drawHalton_η(N_r, S_boot) # New draws for this bootstrap replicate
    res_b = optimize(θ -> neg_msl_boot(θ, idx, η_p_boot, η_sb_boot, η_st_boot), θ_hat, BFGS(), Optim.Options(x_abstol=1e-3)) # Even higher tol here for illustration
    # Check it actually converged; if not, we can either discard this replicate
    if Optim.converged(res_b)
        boot_estimates[b, :] = res_b.minimizer
    else
        println("Bootstrap replicate $b did not converge. Skipping...")  
        boot_estimates[b, :] = NaN * ones(14) # Mark as NaN for later filtering
    end
end
println("Bootstrap done in $(round(time()-t_boot, digits=1)) seconds")
# Drop any non-converged replicates and update B
valid_rows = vec(.!any(isnan.(boot_estimates), dims=2))
boot_estimates_eff = boot_estimates[valid_rows, :]
B_effective = size(boot_estimates_eff, 1)

4. WTP under heterogeneity

Two WTP distributions

With bus and train stops having separate random coefficients, we now have two WTP distributions:

\[ \text{WTP}_{i,\text{bus stop}} = \frac{\beta_{i,\text{stops,b}}}{|\beta_{i,\text{price}}|}, \qquad \text{WTP}_{i,\text{train stop}} = \frac{\beta_{i,\text{stops,t}}}{|\beta_{i,\text{price}}|} \]

We approximate each by drawing \(R = 1{,}000\) triples \((\eta_p, \eta_{sb}, \eta_{st})\) from the estimated distribution.

Analytical cross-check

Since \(\beta_{\text{stops,b}}\) and \(\beta_{\text{price}}\) are independent:

\[ E[\text{WTP}_{\text{bus stop}}] = E[\beta_{\text{stops,b}}] \cdot E \left[\frac{1}{|\beta_{\text{price}}|}\right] = b_{sb} \cdot \exp \left(-b_p + \frac{s_p^2}{2}\right) \]

and similarly \(E[\text{WTP}_{\text{train stop}}] = b_{st} \cdot \exp(-b_p + s_p^2/2)\).

E_WTP_bus_analytic   = b_sb_hat * exp(-b_p_hat + s_p_hat^2 / 2)
E_WTP_train_analytic = b_st_hat * exp(-b_p_hat + s_p_hat^2 / 2)

println("E[WTP bus stop]   (analytical): $(round(E_WTP_bus_analytic*100,   digits=2)) \$/year  " *
        "MC diff: $(round(abs(E_WTP_bus   - E_WTP_bus_analytic)*100,   digits=3)) \$/yr")
println("E[WTP train stop] (analytical): $(round(E_WTP_train_analytic*100, digits=2)) \$/year  " *
        "MC diff: $(round(abs(E_WTP_train - E_WTP_train_analytic)*100, digits=3)) \$/yr")

E[WTP train stop] (analytical): 14.93 $/year  MC diff: 0.001 $/yr

Both should agree to within a few cents, confirming the MC integral has converged. In this case, train stops are a bit off, suggesting low accuracy (which we expected anyway from making \(S\) and \(R\) small for demonstration purposes).

Cross-product marginal effects

In a 3-alternative model (bus, train, car), applying the chain rule to the logit denominator gives the cross-effect of train stops on bus probability, conditional on a draw \(\eta\):

\[ \frac{\partial P_{ib}(\eta)}{\partial \text{stops}_t} = -\beta_{i,\text{st}}(\eta)\cdot P_{ib}(\eta)\cdot P_{it}(\eta) \]

Integrating over the distribution of preferences:

  • In MNL, we have \(-\hat\beta_{st}\cdot P_{ib}^{\text{MNL}}\cdot P_{it}^{\text{MNL}}\)
    • Varies across \(i\) via product of aggregate probabilities only
  • In Mixed Logit, we have \(-E_{\eta}[\beta_{i,st}(\eta)\cdot P_{ib}(\eta)\cdot P_{it}(\eta)]\)
    • Varies across \(i\) via the full joint distribution of \((\beta_{i,st},\,P_{ib}(\eta),\,P_{it}(\eta))\)

Relaxing IIA

So, an advantage of Mixed Logit is that it relaxes the IIA assumption!

  • IIA in MNL: any two commuters with the same \((P_{ib}^{\text{MNL}},\,P_{it}^{\text{MNL}})\) have identical cross-effects, regardless of their demographics.

  • IIA broken in MXL: the expectation \(E_\eta[\beta_{i,st}(\eta)\cdot P_{ib}(\eta)\cdot P_{it}(\eta)]\) is a non-linear average. Even at the same aggregate probabilities, individuals with different \(x_i\) profiles can have different cross-effects because \(\beta_{i,st}\) co-varies differently with the draw-level probabilities.

Let’s see a comparison between the MNL and Mixed Logit cross-effects for our sample of commuters.

5. Confidence intervals for WTP

Two approaches

  1. Bootstrap: resample data, re-estimate \(\hat\theta_b\), recompute \(E[WTP]\) for each \(b\) \(\rightarrow\) empirical distribution of \(E[WTP]\)
    1. Cost: High (\(B\) re-optimizations)
    2. Assumptions: Distribution-free (non-parametric)
  2. Krinsky & Robb (1986, 1990): draw \(\theta^{(k)} \sim N(\hat\theta, \hat V)\), recompute \(E[WTP]\) for each \(k\) \(\rightarrow\) empirical distribution of \(E[WTP]\)
    1. Cost: Low (no re-optimization)
    2. Assumptions: Asymptotic normality of \(\hat\theta\) (parametric)

K&R is much cheaper but relies on normality, which can fail for scale parameters near zero.

Bootstrap CI for E[WTP]

For each bootstrap replicate, we already have \(\hat\theta_b\). Recompute \(E[WTP]\) using the same \(R\) draws.

WTP_bus_boot   = zeros(B_effective)
WTP_train_boot = zeros(B_effective)

Welfare formula under mixed logit

\[ \Delta CS_i = E_\eta\!\left[\frac{1}{|\beta_{i,\text{price}}(\eta)|}\left(\log\!\left(1 + e^{V_{ib}(\eta)} + e^{V_{it}(\eta)}\right) - \log\!\left(1 + e^{V_{ib}(\eta)}\right)\right)\right] \]

Inner expectation over \((\eta_p, \eta_{sb}, \eta_{st})\) by MC (\(R\) draws); outer average over individuals.

Note: we use the train stops random coefficient for \(V_{it}\) in the counterfactual, since the new train system would have train-stop-specific valuation.

Code 
function compute_mean_ΔCS(θ, X_b_c, X_t_c, η_p_d, η_sb_d, η_st_d)
    # Unpack parameters
    R_c  = length(η_p_d); N_c  = size(X_b_c, 1)
    β_b_fx = θ[1:4]; β_t_fx = θ[5:8]
    b_p_c  = θ[9];  s_p_c  = θ[10]
    b_sb_c = θ[11]; s_sb_c = θ[12]
    b_st_c = θ[13]; s_st_c = θ[14]
    ΔCS_i = zeros(N_c) # Initialize vector
    for r in 1:R_c
        β_p_r  = -exp(b_p_c  + s_p_c  * η_p_d[r])
        β_sb_r =      b_sb_c + s_sb_c * η_sb_d[r]
        β_st_r =      b_st_c + s_st_c * η_st_d[r]

        V_ib_r = (X_b_c[:, 1] .* β_b_fx[1]
               .+ X_b_c[:, 2] .* β_p_r
               .+ X_b_c[:, 3] .* β_sb_r    # bus stops × β_bstops
               .+ X_b_c[:, 4] .* β_b_fx[2]
               .+ X_b_c[:, 5] .* β_b_fx[3]
               .+ X_b_c[:, 6] .* β_b_fx[4])

        V_it_r = (X_t_c[:, 1] .* β_t_fx[1]
               .+ X_t_c[:, 2] .* β_p_r
               .+ X_t_c[:, 3] .* β_st_r    # train stops × β_tstops
               .+ X_t_c[:, 4] .* β_t_fx[2]
               .+ X_t_c[:, 5] .* β_t_fx[3]
               .+ X_t_c[:, 6] .* β_t_fx[4])

        log_with    = log.(1.0 .+ exp.(V_ib_r) .+ exp.(V_it_r))
        log_without = log.(1.0 .+ exp.(V_ib_r))

        ΔCS_i .+= (1.0 / abs(β_p_r)) .* (log_with .- log_without)
    end

    ΔCS_i ./= R_c
    return mean(ΔCS_i), ΔCS_i
end;

mean_ΔCS, ΔCS_i = compute_mean_ΔCS(θ_hat, X_b_nr, X_t_nr, η_p_R, η_sb_R, η_st_R)

Takeaways

Key lessons

  1. Random coefficients \(\rightarrow\) intractable likelihood \(\rightarrow\) MSL.
    • Pre-drawing \(\eta\)’s (common random numbers) and the log-sum-exp trick are the two critical implementation details. With 3 random coefficients, we need 3 \(\eta\) matrices.
  2. Bootstrap can get VERY expensive to run, especially with many parameters and simulation draws; parallelization can help, but it’s still a heavy lift.
    • Also, always check convergence in each replicate!
  3. Two paths to CIs for nonlinear functions of \(\hat\theta\):
    • Bootstrap: distribution-free, robust for scale parameters near zero, but slow (\(B\) re-optimizations)
    • Krinsky & Robb: cheap (no re-optimization), relies on asymptotic normality, widely used for reporting WTP in demand studies

Key lessons

  1. Mixed logit unlocks welfare heterogeneity. The \(\Delta\) CS distribution is the economically meaningful output. Some commuters gain more than others
    • Policy analysis that ignores this distribution can miss important equity considerations
  2. More parameters, more structure. Going from 11 (MNL restricted) to 14 (mixed logit) parameters adds interpretable content: separate heterogeneity in bus-stop and train-stop valuations, plus heterogeneity in the price sensitivity.
    • But each extra parameter comes with an identification cost
    • Also, always check that scale parameters don’t collapse to zero

Finally, a warning: we had to simplify A LOT here to make it run in a reasonable time for demonstration purposes. In real research, you would want to use much larger \(S\) and \(R\) for better accuracy, which can make estimation and inference much more computationally intensive.
